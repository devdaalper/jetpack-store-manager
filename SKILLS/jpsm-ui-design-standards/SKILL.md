# JPSM UI Design Standards

Use this skill when implementing or reviewing frontend/UI changes.

## Workflow
1. Load the reference standard:
- `docs/standards/design-standards.md`

2. Enforce design tokens and patterns:
- Reuse CSS variables and component patterns from the reference.
- Avoid ad-hoc colors and one-off component styles.
- Keep desktop/mobile behavior aligned with the documented layout rules.

3. Validate consistency:
- Check new UI follows existing spacing, border radius, and interaction patterns.
- Verify any new component has a clear mapping to existing standards.

## Output
- Implemented UI that matches project standards.
- Short note describing any intentional deviation and why.
